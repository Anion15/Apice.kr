# Security Policy for Apice.kr

## Supported Versions
| Version | Supported          |
|---------|------------------|
| 1.x     | :white_check_mark: |
| 0.x     | :x: Not supported |

## Reporting a Vulnerability
If you discover a security vulnerability in Apice.kr, please report it responsibly:

1. **Do not create a public issue** on GitHub for security issues.
2. Send an email to **admin@apice.kr** with:
   - A clear description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Your contact information

We will respond within **48 hours** and work with you to resolve the issue.

## Security Updates
- All security patches will be communicated through GitHub releases and the repository's release notes.
- Users are advised to upgrade to the latest secure version promptly.

## Reporting Guidelines
- Provide enough details to reproduce the issue.
- Avoid including sensitive data (passwords, keys) in the report.
- Respect responsible disclosure and do not exploit the vulnerability.

## Acknowledgments
We appreciate the community's help in keeping Apice.kr secure. Contributors reporting vulnerabilities may be acknowledged in the release notes or README, with permission.
